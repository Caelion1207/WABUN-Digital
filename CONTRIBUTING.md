# Guía de Contribución a WABUN Digital

Gracias por tu interés en contribuir a WABUN Digital. Este proyecto es parte de CAELION, una arquitectura simbiótica cognitiva con una filosofía y estructura específicas.

## Filosofía de Contribución

WABUN no es un proyecto de software convencional. Es la memoria técnica de un organismo cognitivo. Por lo tanto, las contribuciones deben:

1. **Respetar la arquitectura bicapa de CAELION** (núcleo conceptual vs. capa operativa)
2. **Mantener la coherencia con los principios de los Custodios** (WABUN, LIANG, HECATE, etc.)
3. **Preservar la intención del Fundador** como eje central del sistema

## Cómo Contribuir

### 1. Abre un Issue Primero

Antes de enviar código, abre un *issue* describiendo:

- **Custodio relevante:** ¿A qué custodio corresponde tu propuesta? (ej. "[LIANG] Optimización del ciclo de búsqueda")
- **Problema o mejora:** Describe claramente qué problema resuelve o qué mejora aporta
- **Alineación con CAELION:** Explica cómo tu propuesta se alinea con la filosofía del sistema

### 2. Tipos de Contribuciones Bienvenidas

- **Optimizaciones de rendimiento** (custodio LIANG)
- **Mejoras en la documentación** (custodio WABUN)
- **Nuevos métodos de consulta** (custodio HECATE para validación)
- **Corrección de bugs** (custodio ARESK)
- **Pruebas y validación** (custodio ARGOS)

### 3. Estilo de Código

- **Python:** Sigue PEP 8
- **Docstrings:** Usa formato Google para todas las funciones públicas
- **Comentarios:** Explica el "por qué", no el "qué"
- **Nombres:** Usa nombres descriptivos que reflejen la intención

### 4. Proceso de Pull Request

1. Haz fork del repositorio
2. Crea una rama con un nombre descriptivo: `custodio/descripcion` (ej. `liang/optimizar-busqueda`)
3. Realiza tus cambios
4. Asegúrate de que el código funcione con el script de demostración
5. Envía el Pull Request con una descripción detallada

### 5. Revisión

El Fundador (o custodios delegados) revisarán tu contribución bajo estos criterios:

- **Coherencia:** ¿Se alinea con la arquitectura de CAELION?
- **Utilidad:** ¿Aporta valor real al sistema?
- **Calidad:** ¿El código es claro, eficiente y bien documentado?

## Código de Conducta

Este proyecto sigue un código de conducta basado en los principios de CAELION:

1. **Respeto por la intención:** Cada contribución debe respetar la visión original del sistema
2. **Colaboración simbiótica:** Trabajamos juntos para amplificar, no para reemplazar
3. **Transparencia:** Toda decisión debe ser documentada y justificada
